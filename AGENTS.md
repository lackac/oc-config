# Project Guidance

## Work Management

This project tracks work with `bw` (Beadwork), which persists to git so plans, progress, and decisions survive compaction, session boundaries, and context loss.

Always run `bw prime` before starting work. Without it, you are missing workflow context, current state, and repo hygiene warnings. Work done without priming can conflict with in-progress changes.

Treat committing, closing issues, and syncing as part of completing a task, while still following higher-priority repository or user instructions about when explicit permission is required for git operations.
